# ARGOS

ARGOS is a Deterministic Cognitive Enterprise for AI-assisted financial intelligence, trading research, learning, and education.

This repository begins with EO-001: the repository and project skeleton. It intentionally contains no runtime trading behavior, broker integration, external API connection, or executable trading logic.

## Governing Principles

- Determinism: every process must be reproducible, auditable, version-controlled, and traceable.
- Separation of duties: no organization evaluates its own work.
- Evidence before doctrine: information progresses through Information, Evidence, Validated Knowledge, Doctrine, and Education.
- Auditability: decisions, outputs, recommendations, trades, lessons, and knowledge updates must be traceable to evidence and system state.
- Human control: override, kill switches, approval gates, and operational boundaries are mandatory.
- Modular enterprise design: offices map to services, contracts, interfaces, logs, and tests.
- Safety and financial boundaries: live trading authority requires explicit implementation gates, broker controls, and user approval.

## EO-001 Scope

EO-001 creates the physical repository skeleton only.

In scope:

- Enterprise folder hierarchy
- Placeholder README files
- Python package placeholders
- Repository configuration files
- Structure verification script
- Structure tests

Out of scope:

- Runtime behavior
- Trading logic
- External API connections
- Broker integrations
