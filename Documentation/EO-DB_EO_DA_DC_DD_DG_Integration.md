# EO-DB EO-DA, EO-DC, EO-DD, and EO-DG Integration

EO-DB uses EO-DA authority and invariant registries for authority evidence, EO-DC promotion authority for authoritative truth bridge requirements, EO-DD transaction type registrations for financial owner crossings, and EO-DG route auditing for read-only surface enforcement.

EO-DB owns evidence only. It does not approve truth, coordinate transactions, classify constitutional invariants, or mutate read surfaces.
