# EO-DF Telemetry and Metric Model

`TelemetrySample` captures process resources, runtime structures, communications, broker state, position state, truth/history counts, EO-DD transaction metrics, persistence/checkpoints, cache/cost, and read-model metrics.

Metrics include memory, CPU, threads, tasks, handles, network connections, missions, workflows, tokens, messages, orders, fills, positions, EO-CK enrollments, Performance Truth, Closed Position Truth, Historian, EO-DC decisions, EO-DD journal/outbox, checkpoints, cache, API calls, costs, dashboard requests, Commander requests, latency, digest mismatches, and read-side mutations.

Metric evidence is stored outside financial truth ledgers.
