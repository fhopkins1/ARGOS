# EO-DC Provenance Validation Model

EO-DC validates source identity, authority, timestamp, domain, provenance status, integrity, freshness, observed-vs-derived status, fallback indicators, synthetic indicators, degraded reasons, replay indicators, and test/proof/simulation indicators.

Deterministic rejection codes include provenance missing, unverified, contradictory, synthetic, fallback, degraded, stale, expired, and revoked conditions.

PAPER labels alone are insufficient.
