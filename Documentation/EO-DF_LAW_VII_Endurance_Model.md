# EO-DF LAW VII Endurance Model

EO-DF tracks active workflows, completed workflows, Workflow Tokens, token transfers, active offices, and sleeping offices.

Blocking LAW VII conditions include:
- terminal token remains active
- active office without authorized work
- simultaneous token ownership
- orphan token
- stale-token acceptance

EO-DF consumes EO-DA concepts and does not redefine LAW VII.
