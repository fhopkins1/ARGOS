# EO-DB Canonical Runtime Reachability

EO-DB verifies the runtime bridge path through `CanonicalEnterpriseRuntime` and `CanonicalRuntimeProvider`. Provider construction rejects non-canonical runtimes and live-enabled runtimes.

Certified dynamic reachability must come from the canonical runtime/provider path, not direct office method calls, proof routes, compatibility snapshots, or manual scripts.
