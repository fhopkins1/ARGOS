# EO-DB Runtime Bridge Taxonomy

EO-DB classifies runtime handoffs as `CERTIFIED_PRODUCTION`, `CONDITIONALLY_PRODUCTION`, `PARTIAL`, `MISSING`, `BYPASS`, `DUPLICATE`, `COMPATIBILITY_ONLY`, `PROOF_ONLY`, `SIMULATION_ONLY`, `TEST_ONLY`, `REPLAY_ONLY`, `DEAD`, `DEPRECATED`, `QUARANTINED`, or `UNKNOWN`.

Only certified and explicitly conditional production bridges can count toward operational readiness. Proof, simulation, replay, test, compatibility, and dashboard-only paths are evidence categories, not production certification.
