# EO-DB Required Constitutional Bridge Matrix

The matrix covers Scheduler -> Mission Planner through Assurance Offices -> Commander, including the trade lifecycle, position lifecycle, Performance Truth, Historian, Learning, recovery, replay isolation, cost/API enforcement, and Commander directive boundaries.

The matrix is intentionally honest: bridges that exist but do not yet satisfy all runtime, token, persistence, recovery, EO-DC, EO-DD, and dynamic evidence criteria are classified as `PARTIAL`.
